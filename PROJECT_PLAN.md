# 🏠 자산 및 차량 종합 관리 대시보드 프로젝트 가이드

이 프로젝트는 현재의 자산 상황, 투자 성과 비교 분석 및 보유 차량의 주행 마일리지를 통합 관리할 수 있는 웹 기반 개인 관리 도구입니다. 메인 대시보드 포털을 중심으로 3가지 핵심 기능이 유기적으로 연동됩니다.

---

## 🚀 주요 기능

1. **통합 대시보드 포털 (`index.html`)**:
   - 주택 구매 시뮬레이션, 투자 평가액, 차량별 마일리지 등의 핵심 지표를 요약하여 실시간으로 한눈에 보여줍니다.
   - 각 전용 서비스 페이지로 진입할 수 있는 포털 카드 허브 역할을 수행합니다.

2. **주택 구매 가능성 계산기 (`house.html`)**:
   - 목표 주택 가격, 세금, 중개 수수료 등의 부대비용과 현재 보유 자산(주택 매매 예상가, 현금 등)을 입력하여 필요 대출액을 산출합니다.
   - 금리와 대출 기간에 따른 원리금 균등 상환액, 이자 비용 및 관심 아파트 리스트(참고 시세)를 관리합니다.

3. **투자 성과 및 벤치마크 분석 (`investment.html`)**:
   - 실제 투자 평가금액과 가상 예적금 기회비용을 실시간으로 비교하여 초과 성과(알파, 누적/연환산 수익률)를 계산합니다.
   - Chart.js를 이용한 자산 성장 시각화 그래프를 제공합니다.

4. **차량 마일리지 계산기 (`mileage.html`)**:
   - 보유 차량별 연간 기준 운행 km 대비 현재 날짜 기준 운행 거리를 산출하고, 과달/미달 여부 및 연말 예상 수치를 모니터링합니다.

5. **월간 수입·지출 관리기 (`budget.html`)**:
   - 영기/소땡 님의 공동 수입과 카드/현금별 지출 분류 및 분담 비율을 모니터링합니다.
   - [카드 + 현금] 및 [영기 + 소땡] 합계의 정합성을 실시간 유효성 검사하는 테이블 뷰를 제공합니다.

6. **클라우드 동기화 및 오프라인 백업**:
   - Firestore(Firebase v9 Modular SDK)를 통해 기기 간 데이터를 클라우드에 실시간 동기화합니다.
   - 네트워크 연결이 원활하지 않거나 Firebase가 로드되지 않는 환경에서도 `localStorage` 백업을 통해 정상 작동을 지원합니다.
   - 한국어 환경 전용 날짜 포맷팅 예외 처리를 제공하여 기기 간 동기화 일자(`Invalid Date` 및 일자 잘림 등) 오류를 원천 차단했습니다.

---

## 기술 스택

- **Frontend**: HTML5, Vanilla CSS (style.css), JavaScript (ES6 Modules)
- **Database**: Firebase Firestore (v9 Modular SDK)
- **Visualization**: Chart.js (CDN 라이브러리 연동)
- **Deployment**: GitHub 업로드(Public)

---

## 파일 구조 설명

- **`index.html`** / **`dashboard.js`**: 메인 대시보드 포털 페이지 및 실시간 데이터 바인딩 로직.
- **`house.html`** / **`house.js`**: 주택 구매 가능성 계산기 화면 및 금융/세율 연산 로직.
- **`investment.html`** / **`investment.js`**: 투자 평가 분석 화면, 월별 이자 자본화 복리 연산 및 Chart.js 바인딩.
- **`mileage.html`** / **`mileage.js`**: 차량별 독립 마일리지 설정/측정 화면 및 운행 통계 연산 로직.
- **`budget.html`** / **`budget.js`**: 월간 수입·지출 현금흐름 테이블 관리 화면 및 검증 로직.
- **`style.css`**: 전반적인 글래스모피즘(Glassmorphism) 테마와 반응형 모바일 대응 스타일 시트.
- **`README.md`**: 프로젝트 깃허브 배포 및 실행 소개 문서.

## 주의사항 및 팁

- **로컬 실행 방법**: ES Modules 기술을 사용하므로, 단순히 `.html` 파일을 더블 클릭하여 `file://` 프로토콜로 열면 CORS 정책 오류로 인해 스크립트가 실행되지 않습니다. 반드시 로컬 개발 서버(Live Server 등)를 켜서 `http://localhost` 주소로 실행해 주세요.
- **내비게이션 구조**: 각 서브 페이지에서는 오직 우측 상단의 **[대시보드로]** 버튼을 통해 안전하게 메인 포털 화면으로 회귀하도록 설계되어 있습니다.
- **수동 저장**: 수정한 금융 또는 차량 기록을 기기 간 동기화하려면 화면 상단의 **[기록 저장하기]** 또는 **[저장]** 버튼을 명시적으로 눌러 클라우드에 전송해야 합니다.
